MULTITHER 2.0 ONLINE LIVE SCOREBOARD SETUP
==========================================

What has been added
-------------------
This version already contains:
1. Live Scoreboard panel on the start screen.
2. Live Scoreboard panel on the result screen.
3. Automatic online score submission at the end of each game.
4. Local CSV export kept as backup.
5. Pupil privacy reminder.

Important privacy note
----------------------
When the game is online, avoid using pupils' full real names.
Use pupil codes, class IDs, or nicknames, for example:
- 4A-01
- Pupil 07
- Group Tiger

Step 1: Create Firebase project
-------------------------------
1. Go to Firebase Console.
2. Create a new project, for example: multither-scoreboard.
3. Add a Web App.
4. Firebase will show a firebaseConfig object.

Step 2: Create Realtime Database
--------------------------------
1. In Firebase Console, open Build > Realtime Database.
2. Create Database.
3. Choose a region near Malaysia/Singapore if available.
4. Start in test mode only for first testing.

Step 3: Paste config into firebase-config.js
-------------------------------------------
Open firebase-config.js and replace the placeholder values with your own Firebase web config.
Then change:

window.MULTITHER_SCOREBOARD_OPTIONS = {
  enabled: false,
  databasePath: "multither2_scoreboard",
  maxRows: 10
};

into:

window.MULTITHER_SCOREBOARD_OPTIONS = {
  enabled: true,
  databasePath: "multither2_scoreboard",
  maxRows: 10
};

Step 4: Suggested temporary classroom rules
------------------------------------------
These rules are for controlled classroom testing only.
Do not use full pupil names.

{
  "rules": {
    "multither2_scoreboard": {
      ".read": true,
      "$scoreId": {
        ".write": "newData.exists()",
        ".validate": "newData.hasChildren(['attemptId','createdAt','pupilName','className','modeLabel','levelLabel','accuracy','speedPerMinute','score']) && newData.child('score').isNumber() && newData.child('accuracy').isNumber() && newData.child('pupilName').isString() && newData.child('className').isString()"
      }
    }
  }
}

For stronger security, use Firebase Authentication or ask a developer to add a teacher-only admin layer.

Step 5: Upload to GitHub Pages
------------------------------
Upload all files, including:
- index.html
- style.css
- game.js
- firebase-config.js
- background.jpg
- gameLogo.png
- ICON.ico
- SlitheringNumbers.mp3

Open the GitHub Pages link.
If Firebase is correctly connected, the scoreboard status will show:
Live scoreboard connected

How it works
------------
When a pupil finishes a game, the game sends the final score online.
Other devices that open the same GitHub Pages game will see the updated ranking.

CSV export still works
----------------------
The Export CSV button still exports local results saved in that browser/device.
This is useful as a backup when Wi-Fi is weak.

MULTITHER 2.0 v52 - Teacher/Admin Login Troubleshooting

If Teacher/Admin Login fails, this version shows the exact Firebase error code on screen.

Most common fixes:
1. auth/invalid-credential or auth/wrong-password
   - Firebase does not let you view the old password.
   - Go to Firebase Authentication > Users > select the account > Reset password / change password.

2. auth/operation-not-allowed
   - Go to Firebase Authentication > Sign-in method > Email/Password > Enable > Save.

3. auth/unauthorized-domain
   - Go to Firebase Authentication > Settings > Authorized domains.
   - Add: catherinekiu.github.io
   - Do not include https:// or /* in this section.

4. Referer blocked / API key blocked
   - Go to Google Cloud Console > APIs & Services > Credentials.
   - Click Browser key (auto created by Firebase).
   - Application restrictions: Websites.
   - Add: https://catherinekiu.github.io/*
   - Add exact game URL if needed.
   - For testing, set API restrictions to Don't restrict key. If it works, later add required APIs carefully.

5. If testing locally from file:///C:/...
   - Teacher/Admin Firebase login may fail after key restrictions.
   - Test Teacher/Admin Login from the GitHub Pages link, not from a local file.

Important:
Pupils do not need login. Teacher/Admin login is only for protected online CSV export.
